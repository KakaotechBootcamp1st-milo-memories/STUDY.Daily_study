# 개발자 채용 절차
서류 -> 코딩 테스트 -> 기술 면접, 시스템 디자인 면접 -> 기업 문화 적합도 면접

## 서류
- 지원자의 이력서와 포트폴리오를 통해 첫인상을 형성하는 단계
- 지원자의 기술 스택, 프로젝트 경험 등을 평가
- 서류 전형을 통해 적합한 지원자를 선별하고, 면접 등 이후 절차에서 시간을 효율적으로 사용
## 코딩 테스트
- 지원자의 문제 해결 능력과 알고리즘적 사고를 평가
- 주어진 시간 내에 문제를 정확하게 풀어낼 수 있는지를 확인
- 지원자의 기술적 기초와 논리적 사고력을 평가하여, 실무에서의 문제 해결 능력을 미리 판단
## 기술 면접, 시스템 디자인 면접
- 지원자의 기술적 깊이와 실무 적용 능력을 평가
- 구체적인 기술적 질문과 기업에 따라 라이브 코딩 테스트 시행
- 대규모 시스템 설계 능력을 평가
- 확장성, 안정성, 효율성 등을 고려한 설계 능력을 확인
## 기업 문화 적합도 면접
- 지원자가 회사의 가치관과 문화에 얼마나 잘 맞는지를 평가
- 지원자의 성향, 대인 관계 능력, 회사의 비전과의 일치도를 중점적으로 봄
- 팀의 일원으로서 지원자가 얼마나 잘 적응하고, 조직 내에서 긍정적인 영향을 미칠 수 있는지를 평가

# 채용 절차 준비 전략
## 1. 이력서, 포트폴리오
- 이력서 : 채용 담당자에게 지원자의 기본 정보를 전달
- 포트폴리오
- 구체적인 프로젝트와 실무 능력을 시각적으로 증명
- 실제 작업물을 통해 기술 역량을 보여줌
### 이력서 작성
- 개인 정보: 이름, 연락처, 이메일 등 필수 정보는 명확하고 깔끔하게 기재
- 학력: 최신 학력부터 나열, 전공과 관련된 주요 과목 강조
- 지원 동기, 목표: 지원하는 포지션에 맞춘 지원 동기와 커리어 목표 명시
- 기술 스택
  - 사용 가능한 프로그래밍 언어, 프레임워크, 도구 나열
  - 각 기술에 대한 숙련도가 표시 되면 좋음
  - 표, 아이콘을 활용해 시각적으로 깔끔하게 표기
- 인턴십, 프로젝트 경험
  - 사용한 기술, 기여도, 해결한 문제들을 명시
  - STAR 방법론(상황, 과제, 행동, 결과)을 활용해 기술
### 포트폴리오 작성
- 자기소개: 간단한 자기소개, 개발자로서의 철학, 목표
- 기술 스택: 사용할 수 있는 프로그래밍 언어, 프레임워크, 툴에 대한 숙련도 표기
- 기술 블로그,프로필 링크
  - 운영중인 기술 관련 블로그 링크, 포스팅 자료
  - Github 등 대외적인 활동 사례 포함
- 대표 프로젝트
  - 어필하면 좋을 프로젝트 3~5개 선정
  - 각 프로젝트의 목표, 과정, 결과 설명
  - 사용한 기술 스택과 도구 설명
  - 문제 해결 과정, 성과 (기여도) 강조
  - 프로젝트 흐름과 결과를 한눈에 보여주는 다이어그램
- 프로젝트 외의 활동
  - 참여한 해커톤 및 경진대회
  - 교육 및 학습 경험
  - 연구 및 논문
### 실수하기 쉬운 부분들
- 이력서 관련
  - 오타 및 문법 오류
  - 연락처 오류
  - 과장된 기술 능력
  - 너무 긴 이력서
- 포트폴리오 관련
  - 너무 많은 프로젝트 나열
  - 복잡한 디자인
  - 링크 오류
  - 파일 형식 오류

### 실제 준비 전략
#### LeetCode
- LeetCode를 활용한 알고리즘 연습
- 알고리즘 문제의 카테고리를 보지 않고, Easy부터 Hard까지 점차적으로 문제를 풀어나감
- 문제를 이해하고 해결하기 위해 충분한 사고와 노트 이후 코딩을 시작하기
- 시간이 너무 오래 걸리면 솔루션을 참고하고, 같은 실수를 반복하지 않기 위해 솔루션에서 배운 아이디어를 정리
- 기수 정렬, 유니언 파인드와 같은 자주 나오지 않는 알고리즘을 학습해 완성도 더하기
#### CodeForces
- CodeForces를 활용한 알고리즘 연습
- 실제 콘테스트에서 문제를 풀며, 라이브 코딩 테스트 환경을 경험
- 콘테스트에서 풀지 못한 문제는 꼭 시간을 내서 풀어봄
- 자신이 작성한 코드를 다시 검토하고, 코드의 시공간 복잡도를 개선
#### 환경 설정
- IDE 및 편집기 설정
  - 평소 익숙한 IDE 또는 텍스트 편집기를 사용해 실수를 줄임
  - 자동 완성, 코드 스니펫, 디버깅 툴 등 유용한 기능들을 미리 설정
- 코드 템플릿 준비
  - 자주 사용하는 함수나 자료구조의 템플릿을 준비해 시간 절약
  - BFS, DFS, 정렬 알고리즘 등 자주 사용하는 알고리즘의 기본 코드 준비
  - 사용할 수 없는 환경도 있을 수 있음
- 인터넷 연결 및 환경 체크
  - 테스트 전에 인터넷 연결 상태를 확인
  - 예비 배터리, 백업 기기 준비
#### 실수하기 쉬운 부분
- Off-by-one Error
- 입출력 형식 오류
- 시간 관리 실패
- 문제 이해 부족
- 엣지 케이스 고려 부족
- 커다란 디버깅 단위
## 기술 면접, 시스템 디자인 면접
- 기술 면접
  - 지원자의 CS 기초 지식과 실무 역량 평가
  - 문제 해결 능력과 사고 과정을 직접 구술하며 확인
- 시스템 디자인 면접
  - 복잡한 시스템을 설계하는 능력 평가
  - 시스템의 확장성, 신뢰성, 성능 최적화 등을 고려하는 능력 확인
- CS 기초 지식 강화
  - 강의, 책, 온라인 코스 등을 활용해 필요 CS 지식 강화
- 면접 질문 예측 및 연습
  - 자주 묻는 CS 질문과 답변을 미리 준비
  - 지원한 포지션과 연관된 실무에서의 상황을 가정한 질문에 답변 연습
- 커뮤니케이션 연습
  - 복잡한 Cs 개념을 쉽게 설명하는 연습
  - 모의 면접을 진행하여 실전 감각 향상
- 기본적인 시스템 설계 개념 학습
  - 클라이언트-서버 모델, 마이크로서비스 아키텍처, 캐싱, 데이터베이스 샤딩 등 시스템 설계 개념 학습
- 기능적 및 비기능적 요구 사항 고려
  - 기능적 요구 사항: 시스템이 무엇을 해야 하는지 명확히 정의
  - 비기능적 요구 사항: 성능, 보안, 확장성 등 설계 시 고려해야 할 비기능적 요구 사항 준비
- 트레이드오프 이해
  - 성능 vs 비용
  - 유연성 vs 복잡성
### 실제 준비 전략 - 기술 면접
- CS 지식 학습
  - 05, 네트워크, 데이터베이스, 프로그래밍 언어 등 다양한 분야의 CS 지식 학습
  - 암기보다는 개념을 명확히 이해하고 응용할 수 있는 능력 배양
- 2Depth 이상 답변 훈련
  - 꼬리물기 질문에 대비해 2 Depth 이상의 심층 답변 연습
- 그룹 스터디 활용
  - 그룹 스터디를 통해 다양한 시각에서 CS 지식 베이스 확장
  - 스터디 멤버들로부터 피드백을 받아 부족한 부분 보완
### 실제 준비 전략 - 시스템 디자인 면접
- 시스템 디자인 인터뷰 방식 학습
  - 기능적, 비기능적 요구 사항을 인터뷰어와 대화하며 정의하는 방법을 학습
  - 설계를 다 하고 답하는 것이 아닌 설계 과정에서의 의사 결정 과정을 명확히 설명
- 유명한 시스템 디자인 질문 분석
  - URL 단축기, 유일 ID 생성기 등의 유명한 시스템 디자인 문제를 깊이 있게 분석
- 확장성 고려 연습
  - 시스템이 확장되면서 발생할 수 있는 문제들을 예측하고, 대응 방안을 연습
- Mock 인터뷰 진행
  - 여러 가지 주제의 시스템 디자인 질문을 이용해 모의 면접 진행

- 컬처핏 면접 예상 질문 준비
  - 자주 묻는 질문 분석
  - 조사한 기업 문화 지식베이스로 구체적인 답변 준비
